#htb #web #http 

HTTP поддерживает несколько методов доступа к ресурсу. В протоколе HTTP несколько методов запроса позволяют браузеру отправлять на сервер информацию, формы или файлы. Эти методы используются, в частности, для того, чтобы сообщить серверу, как обрабатывать отправленный нами запрос и как отвечать.

Мы видели различные HTTP-методы, используемые в HTTP-запросах, которые мы тестировали в предыдущих разделах. С cURL, если мы используем `-v` для предварительного просмотра полного запроса, первая строка содержит HTTP-метод (например, `GET / HTTP/1.1`), в то время как с browser devtools HTTP-метод показан в `Method` столбце. Кроме того, заголовки ответа также содержат код состояния HTTP, который указывает на результат обработки нашего HTTP-запроса.

## Методы запроса

Ниже приведены некоторые из наиболее часто используемых методов:

|**Способ**|**Описание**|
|---|---|
|`GET`|Запрашивает определённый ресурс. Дополнительные данные можно передать на сервер с помощью строки запроса в URL (например, `?param=value`).|
|`POST`|Отправляет данные на сервер. Может обрабатывать различные типы входных данных, такие как текст, PDF-файлы и другие формы двоичных данных. Эти данные добавляются в тело запроса после заголовков. Метод POST обычно используется для отправки информации (например, форм/логинов) или загрузки данных на веб-сайт, таких как изображения или документы.|
|`HEAD`|Запрашивает заголовки, которые были бы возвращены при отправке на сервер запроса GET. Запрос не возвращает тело запроса и обычно используется для проверки длины ответа перед загрузкой ресурсов.|
|`PUT`|Создаёт новые ресурсы на сервере. Использование этого метода без надлежащего контроля может привести к загрузке вредоносных ресурсов.|
|`DELETE`|Удаляет существующий ресурс на веб-сервере. При отсутствии надлежащей защиты может привести к отказу в обслуживании (DoS) из-за удаления важных файлов на веб-сервере.|
|`OPTIONS`|Возвращает информацию о сервере, например о поддерживаемых им методах.|
|`PATCH`|Применяет частичные изменения к ресурсу в указанном месте.|

В списке приведены лишь некоторые из наиболее часто используемых методов HTTP. Доступность того или иного метода зависит от сервера, а также от конфигурации приложения. Полный список методов HTTP можно найти по этой [ссылке](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods).

**Примечание:** большинство современных веб-приложений в основном используют методы `GET` и `POST` . Однако любое веб-приложение, использующее REST API, также использует методы `PUT` и `DELETE`, которые применяются для обновления и удаления данных в конечной точке API соответственно. Подробнее см. в модуле [Введение в веб-приложения](https://academy.hackthebox.com/module/details/75).

## Коды состояния

Коды состояния HTTP используются для информирования клиента о статусе его запроса. HTTP-сервер может возвращать пять классов кодов состояния:

|**Класс**|**Описание**|
|---|---|
|`1xx`|Предоставляет информацию и не влияет на обработку запроса.|
|`2xx`|Возвращается в случае успешного выполнения запроса.|
|`3xx`|Возвращается, когда сервер перенаправляет клиента.|
|`4xx`|Указывает на некорректные запросы `from the client`. Например, на запрос несуществующего ресурса или запрос в неправильном формате.|
|`5xx`|Возвращается, если возникает какая-либо проблема `with the HTTP server` сама по себе.|

Ниже приведены наиболее распространённые примеры для каждого из вышеперечисленных классов кодов состояния HTTP:

|**Код**|**Описание**|
|---|---|
|`200 OK`|Возвращается при успешном запросе, а тело ответа обычно содержит запрошенный ресурс.|
|`302 Found`|Перенаправляет клиента на другой URL. Например, перенаправляет пользователя на панель управления после успешного входа в систему.|
|`400 Bad Request`|Возвращается при обнаружении некорректных запросов, например запросов без терминаторов строк.|
|`403 Forbidden`|Означает, что у клиента нет соответствующего доступа к ресурсу. Также может возвращаться, когда сервер обнаруживает вредоносные данные, введенные пользователем.|
|`404 Not Found`|Возвращается, когда клиент запрашивает ресурс, которого нет на сервере.|
|`500 Internal Server Error`|Возвращается, когда сервер не может обработать запрос.|

Полный список стандартных кодов состояния HTTP можно найти по этой [ссылке](https://developer.mozilla.org/en-US/docs/Web/HTTP/Status). Помимо стандартных кодов HTTP, различные серверы и провайдеры, такие как [Cloudflare](https://support.cloudflare.com/hc/en-us/articles/115003014432-HTTP-Status-Codes) или [AWS](https://docs.aws.amazon.com/AmazonSimpleDB/latest/DeveloperGuide/APIError.html), используют собственные коды.
